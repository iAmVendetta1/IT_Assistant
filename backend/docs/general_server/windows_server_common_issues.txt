Title: Common Windows Server Issues and Recommended Resolutions

Overview:
This document outlines frequently encountered issues on Windows Server environments (2012 R2, 2016, 2019, 2022). These steps are intended for helpdesk and systems technicians performing first‑line and second‑line troubleshooting.

1. Group Policy Not Applying
Symptoms:
- Users receive outdated settings
- gpupdate /force shows errors
- Event Viewer logs 1058 or 1030

Resolution:
- Verify SYSVOL and NETLOGON shares are accessible
- Check for replication issues using `dfsrdiag` or `repadmin /replsummary`
- Ensure the workstation and domain controller clocks are in sync
- Validate permissions on the GPO folder structure
- Run `gpresult /h report.html` to confirm scope and filtering

2. DNS Resolution Failures
Symptoms:
- Intermittent name resolution
- Slow logons
- Applications unable to locate servers

Resolution:
- Ensure the server points ONLY to internal DNS servers
- Restart the DNS Client service
- Clear DNS cache: `ipconfig /flushdns`
- Check for stale or duplicate A records
- Validate forwarders and root hints

3. High CPU Usage by svchost.exe
Symptoms:
- Server becomes sluggish
- Task Manager shows svchost.exe consuming high CPU

Resolution:
- Identify the service group using Task Manager → “Go to Service(s)”
- Common culprits: Windows Update, WMI, DNS, DHCP
- Restart the problematic service
- Check for stuck Windows Updates
- Rebuild WMI repository if necessary

4. Windows Update Failures
Symptoms:
- Updates stuck at “Downloading” or “Installing”
- Error codes such as 0x800f081f or 0x80244022

Resolution:
- Run Windows Update Troubleshooter
- Reset update components:
  net stop wuauserv
  net stop bits
  rename C:\Windows\SoftwareDistribution SoftwareDistribution.old
  net start wuauserv
  net start bits
- Ensure the server has internet access or WSUS connectivity
- Check Event Viewer → WindowsUpdateClient logs

5. Disk Space Running Low on C:
Symptoms:
- Server alerts for low disk space
- Services fail to start
- Updates cannot install

Resolution:
- Clear temp files and old Windows Update cache
- Move paging file to another volume if appropriate
- Check for oversized log files (IIS, SQL, DFSR)
- Run `cleanmgr /sageset:1` and `cleanmgr /sagerun:1`
- Expand the virtual disk if running on a hypervisor

6. Active Directory Replication Issues
Symptoms:
- Users unable to authenticate
- GPO failures
- Event IDs 1311, 1865, 2042

Resolution:
- Run `repadmin /replsummary` to identify failing DCs
- Check site/subnet configuration in AD Sites and Services
- Validate DNS SRV records
- Ensure DCs can communicate over required ports (RPC, LDAP, SMB)
- Restart NTDS service if safe to do so

7. RDP Connection Failures
Symptoms:
- “Remote Desktop can’t connect to the remote computer”
- Black screen after login
- Session immediately disconnects

Resolution:
- Confirm RDP is enabled in System Properties
- Check Windows Firewall rules for RDP (TCP 3389)
- Restart Remote Desktop Services
- Verify licensing mode if using RDS
- Update GPU drivers on virtualized servers

8. DHCP Not Issuing Addresses
Symptoms:
- Clients receive APIPA addresses (169.254.x.x)
- DHCP console shows no active leases

Resolution:
- Ensure the DHCP server is authorized in Active Directory
- Confirm scope is active and not out of addresses
- Check for rogue DHCP servers on the network
- Validate that the DHCP service is running
- Review Event Viewer → DHCP Server logs

============================================================
SECTION 1 — SERVICES NOT STARTING
============================================================

ISSUE: Critical Windows services fail to start (DNS, DHCP, AD DS, etc.)
CAUSES:
- Missing dependencies.
- Incorrect service account permissions.
- Corrupt system files.

RESOLUTION:
1. Check service status:
   - Run: services.msc
   - Look for services stuck on “Starting” or “Stopped”.

2. Check dependencies:
   - Right‑click service → Properties → Dependencies tab.

3. Start service via PowerShell:
   - Start-Service -Name <ServiceName>
   - Check status:
     Get-Service <ServiceName>

4. Check Event Viewer:
   - Event Viewer → Windows Logs → System.
   - Look for Service Control Manager errors.

5. Run SFC/DISM:
   - Command Prompt (Admin):
     sfc /scannow
     DISM /Online /Cleanup-Image /RestoreHealth

============================================================
SECTION 2 — DNS SERVER ISSUES
============================================================

ISSUE: DNS not resolving internal names.
CAUSES:
- DNS service stopped.
- Corrupt zone data.
- Forwarders misconfigured.

RESOLUTION:
1. Restart DNS service:
   - services.msc → DNS Server → Restart.

2. Check zone status:
   - DNS Manager → Forward Lookup Zones.
   - Ensure zone is not paused or in error state.

3. Test DNS:
   - nslookup <hostname>
   - nslookup <hostname> <DNS server IP>

4. Verify forwarders:
   - DNS Manager → Server Properties → Forwarders tab.

5. Clear DNS cache:
   - ipconfig /flushdns
   - dnscmd /clearcache

============================================================
SECTION 3 — ACTIVE DIRECTORY ISSUES
============================================================

ISSUE: Domain controller replication failing.
CAUSES:
- SYSVOL not shared.
- DNS issues.
- AD DS service issues.

RESOLUTION:
1. Check replication:
   - Command Prompt:
     repadmin /replsummary
     repadmin /showrepl

2. Check SYSVOL/NETLOGON:
   - Run:
     net share
   - Ensure SYSVOL and NETLOGON are listed.

3. Restart AD DS services:
   - services.msc → Active Directory Domain Services → Restart.

4. Check DC health:
   - Command Prompt:
     dcdiag /v

------------------------------------------------------------

ISSUE: Cannot join workstation to domain.
CAUSES:
- DNS misconfigured.
- Time skew.
- Network path issues.

RESOLUTION:
1. Ensure workstation DNS points ONLY to domain controller.
2. Check time sync:
   - Run:
     w32tm /query /status
   - Time difference must be < 5 minutes.

3. Test domain connectivity:
   - ping <domain controller>
   - nslookup domain.local

============================================================
SECTION 4 — DHCP SERVER ISSUES
============================================================

ISSUE: Clients not receiving IP addresses.
CAUSES:
- DHCP service stopped.
- Scope exhausted.
- Server not authorized.

RESOLUTION:
1. Check DHCP service:
   - services.msc → DHCP Server → Start.

2. Verify server authorization:
   - DHCP Console → Right‑click server → Authorize.

3. Check scope:
   - Ensure available IPs remain.
   - Expand scope if needed.

4. Check Event Viewer:
   - Applications and Services Logs → Microsoft → Windows → DHCP‑Server.

============================================================
SECTION 5 — FILE SERVER ISSUES
============================================================

ISSUE: Users cannot access shared folders.
CAUSES:
- NTFS permissions incorrect.
- Share permissions incorrect.
- Offline domain controller.

RESOLUTION:
1. Check share permissions:
   - Right‑click folder → Properties → Sharing → Advanced Sharing → Permissions.

2. Check NTFS permissions:
   - Properties → Security tab.

3. Test access:
   - \\servername\sharename

4. Restart Server service:
   - services.msc → Server → Restart.

============================================================
SECTION 6 — REMOTE DESKTOP (RDP) ISSUES
============================================================

ISSUE: Cannot RDP into server.
CAUSES:
- RDP disabled.
- Firewall blocking port 3389.
- NLA authentication issues.

RESOLUTION:
1. Enable RDP:
   - Server Manager → Local Server → Remote Desktop → Enable.

2. Allow firewall rule:
   - PowerShell:
     Enable-NetFirewallRule -DisplayGroup "Remote Desktop"

3. Check RDP service:
   - services.msc → Remote Desktop Services → Restart.

4. Disable NLA temporarily (if needed):
   - System Properties → Remote → uncheck “Allow connections only with NLA”.

============================================================
SECTION 7 — WINDOWS UPDATE FAILURES
============================================================

ISSUE: Updates failing to install.
CAUSES:
- Corrupt update cache.
- Damaged system files.

RESOLUTION:
1. Reset Windows Update components:
   - Command Prompt (Admin):
     net stop wuauserv
     net stop bits
     del /Q C:\Windows\SoftwareDistribution\*
     net start wuauserv
     net start bits

2. Run SFC/DISM:
   - sfc /scannow
   - DISM /Online /Cleanup-Image /RestoreHealth

3. Check Windows Update log:
   - Event Viewer → Applications and Services Logs → Microsoft → Windows → WindowsUpdateClient.

============================================================
SECTION 8 — STORAGE / DISK ISSUES
============================================================

ISSUE: Disk space low or server running out of space.
CAUSES:
- Large log files.
- Windows Update cache.
- Old backups.

RESOLUTION:
1. Run Disk Cleanup:
   - Server Manager → Add Features → Desktop Experience (if needed).
   - Cleanmgr.exe → select drive.

2. Clear Windows Update cache:
   - del /Q C:\Windows\SoftwareDistribution\Download\*

3. Check large folders:
   - PowerShell:
     Get-ChildItem C:\ -Recurse | Sort Length -Descending | Select -First 20

------------------------------------------------------------

ISSUE: Disk errors or failing drive.
CAUSES:
- Bad sectors.
- RAID degradation.

RESOLUTION:
1. Run CHKDSK:
   - chkdsk C: /f /r

2. Check RAID controller logs.

3. Replace failing drive if SMART errors appear.

============================================================
SECTION 9 — PERFORMANCE ISSUES
============================================================

ISSUE: Server slow or unresponsive.
CAUSES:
- High CPU usage.
- Memory leaks.
- Disk bottlenecks.

RESOLUTION:
1. Check Task Manager:
   - Identify processes using high CPU/RAM.

2. Check disk performance:
   - Resource Monitor → Disk tab.

3. Restart problematic services (IIS, SQL, etc.).

4. Check Event Viewer for warnings/errors.

============================================================
SECTION 10 — CERTIFICATE / SSL ISSUES
============================================================

ISSUE: HTTPS services not loading (IIS, RDS Gateway, etc.)
CAUSES:
- Expired certificate.
- Binding missing.
- Incorrect private key.

RESOLUTION:
1. Check certificate:
   - mmc → Add snap‑in → Certificates → Local Computer → Personal → Certificates.

2. Check IIS binding:
   - IIS Manager → Sites → Bindings → HTTPS → select correct certificate.

3. Reimport certificate with private key (.pfx).
