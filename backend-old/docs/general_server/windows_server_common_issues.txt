Title: Common Windows Server Issues and Recommended Resolutions

Overview:
This document outlines frequently encountered issues on Windows Server environments (2012 R2, 2016, 2019, 2022). These steps are intended for helpdesk and systems technicians performing first‑line and second‑line troubleshooting.

1. Group Policy Not Applying
Symptoms:
- Users receive outdated settings
- gpupdate /force shows errors
- Event Viewer logs 1058 or 1030

Resolution:
- Verify SYSVOL and NETLOGON shares are accessible
- Check for replication issues using `dfsrdiag` or `repadmin /replsummary`
- Ensure the workstation and domain controller clocks are in sync
- Validate permissions on the GPO folder structure
- Run `gpresult /h report.html` to confirm scope and filtering

2. DNS Resolution Failures
Symptoms:
- Intermittent name resolution
- Slow logons
- Applications unable to locate servers

Resolution:
- Ensure the server points ONLY to internal DNS servers
- Restart the DNS Client service
- Clear DNS cache: `ipconfig /flushdns`
- Check for stale or duplicate A records
- Validate forwarders and root hints

3. High CPU Usage by svchost.exe
Symptoms:
- Server becomes sluggish
- Task Manager shows svchost.exe consuming high CPU

Resolution:
- Identify the service group using Task Manager → “Go to Service(s)”
- Common culprits: Windows Update, WMI, DNS, DHCP
- Restart the problematic service
- Check for stuck Windows Updates
- Rebuild WMI repository if necessary

4. Windows Update Failures
Symptoms:
- Updates stuck at “Downloading” or “Installing”
- Error codes such as 0x800f081f or 0x80244022

Resolution:
- Run Windows Update Troubleshooter
- Reset update components:
  net stop wuauserv
  net stop bits
  rename C:\Windows\SoftwareDistribution SoftwareDistribution.old
  net start wuauserv
  net start bits
- Ensure the server has internet access or WSUS connectivity
- Check Event Viewer → WindowsUpdateClient logs

5. Disk Space Running Low on C:
Symptoms:
- Server alerts for low disk space
- Services fail to start
- Updates cannot install

Resolution:
- Clear temp files and old Windows Update cache
- Move paging file to another volume if appropriate
- Check for oversized log files (IIS, SQL, DFSR)
- Run `cleanmgr /sageset:1` and `cleanmgr /sagerun:1`
- Expand the virtual disk if running on a hypervisor

6. Active Directory Replication Issues
Symptoms:
- Users unable to authenticate
- GPO failures
- Event IDs 1311, 1865, 2042

Resolution:
- Run `repadmin /replsummary` to identify failing DCs
- Check site/subnet configuration in AD Sites and Services
- Validate DNS SRV records
- Ensure DCs can communicate over required ports (RPC, LDAP, SMB)
- Restart NTDS service if safe to do so

7. RDP Connection Failures
Symptoms:
- “Remote Desktop can’t connect to the remote computer”
- Black screen after login
- Session immediately disconnects

Resolution:
- Confirm RDP is enabled in System Properties
- Check Windows Firewall rules for RDP (TCP 3389)
- Restart Remote Desktop Services
- Verify licensing mode if using RDS
- Update GPU drivers on virtualized servers

8. DHCP Not Issuing Addresses
Symptoms:
- Clients receive APIPA addresses (169.254.x.x)
- DHCP console shows no active leases

Resolution:
- Ensure the DHCP server is authorized in Active Directory
- Confirm scope is active and not out of addresses
- Check for rogue DHCP servers on the network
- Validate that the DHCP service is running
- Review Event Viewer → DHCP Server logs

End of Document.
